Multi-Agent 投資決策架構：讓 AI 像基金團隊一樣思考

一個人做投資決策容易有盲點，基金公司因此設置了不同角色的分析師。那如果 AI 也能這樣分工合作呢？這篇分享用 LangGraph 建構 Multi-Agent 系統的實戰經驗。

架構設計

我們設計了五個 Agent，各司其職：

  • 技術分析師 — 計算技術指標，判斷趨勢
  • 新聞情緒分析師 — 爬取即時新聞，量化市場情緒
  • 基本面分析師 — 分析財報數據，評估公司價值
  • 多空辯論員 — 整合以上觀點，正反辯論
  • 交易員 — 根據辯論結論，產出最終買賣訊號

為什麼用 LangGraph？

比起 CrewAI 或 AutoGen，LangGraph 給了我們更細粒度的流程控制。每個 Agent 之間的資料傳遞、條件分支、錯誤處理都可以用 Graph 的方式清楚定義。對金融場景來說，「可控性」比「自主性」更重要。

MCP 串接 TEJ 數據

透過 MCP（Model Context Protocol）工具，每個 Agent 可以直接呼叫 TEJ API 取得即時數據。這讓整個系統不只是在「聊天」，而是真的在做數據驅動的決策。

回測驗證

最後我們把產出的訊號接入 Zipline 回測引擎，在台股歷史資料上跑了三年回測。雖然不能公開具體績效數字，但結果讓我們對這個架構的可行性有了信心。

反思

Multi-Agent 系統最大的挑戰不是讓每個 Agent 變聰明，而是設計好它們之間的「溝通協議」。誰先說話、誰有最終決定權、衝突怎麼解決——這些流程設計決定了系統的上限。
